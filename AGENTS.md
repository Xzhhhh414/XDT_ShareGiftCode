# Project Agent Instructions

本项目主要由 AI 协助开发。任何代理或开发者修改本仓库内容时，必须遵守以下规则。

## 产品文档同步

- 修改任何产品行为、界面文案、交互流程、数据结构、样式布局、规则判断或开发阶段计划时，必须同步更新产品文档。
- 当前产品文档入口是 `README.md` 和 `docs/PLAN.md`。
- `README.md` 记录当前产品定位、核心规则、页面结构和 MVP 范围。
- `docs/PLAN.md` 记录阶段计划、已完成进度、后续演进和需要保留的产品决策。
- 如果一次修改只涉及纯格式化、注释调整或不改变用户可见行为的内部整理，可以不更新产品文档，但提交说明或交付说明里必须明确说明“无需更新产品文档”的原因。
- 修改完成前需要检查代码、数据和产品文档是否一致，不能让实现和文档描述互相冲突。

## 开发原则

- 页面首屏直接展示兑换码工具，不做营销落地页。
- 使用同一套数据和组件，通过响应式布局适配 PC Web 和 Mobile Web。
- 只展示公开来源中的兑换码信息，不自行生成兑换码。
- 已失效或已过期兑换码不展示给玩家，但数据可保留用于去重和追溯。
- 所有时间统一按 Asia/Shanghai 展示。
